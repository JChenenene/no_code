# 适用场景

该模式库用于指导官网首页、产品发布页、定价页、案例页等页面的首屏与转化区设计。它不是一整页模板，而是解决“首屏怎么起势、CTA 怎么排布、哪些元素真正影响转化”的通用结构规则。

# 页面目标

Hero 的核心目标是快速建立品牌记忆、说明价值并给出下一步动作。CTA 的目标是把决策负担降到最低，让用户不用思考“点哪里”。二者必须协同工作，而不是视觉独立存在。

# 推荐信息架构

高质量 Hero 常见结构为：小标签或定位、主标题、副标题、主 CTA、次 CTA、视觉主图或界面预览、可信补充。CTA 区不要放太多按钮，优先一主一辅。若页面更偏咨询型，可改成一个主按钮加一个文字链接；若偏产品体验型，可用“立即体验 + 查看演示”的组合。

# 必备模块

- 品牌识别：Logo、产品名、版本标识或品类标签
- 价值表达：一句强标题加一段结果导向副标题
- CTA 组合：主动作和辅助动作
- 可信补充：客户数量、上线速度、支持对象、成果数字
- 主视觉：界面预览、抽象图形或数据型视觉

# 视觉与交互重点

首屏主标题应是最大视觉焦点，CTA 次之，视觉主图负责承接品牌气质。CTA 按钮要有明显主次，主按钮使用高对比实体色或发光描边，次按钮则用更克制的边框或文字按钮。交互只做轻微强化，例如 hover 高亮、描边发光、卡片浮起，不要让 Hero 看起来像游戏启动页。

# 文案语气与禁忌

标题建议围绕结果和对象来写，例如“让企业站点从需求到上线更快完成”“为 AI 产品提供统一的品牌级页面生成体验”。CTA 用词应明确，如“立即体验”“预约演示”“开始创建”。避免使用“点击了解更多”这种弱动作，也不要在首屏出现过多形容词而没有明确结果。

# 响应式与实现边界

移动端 Hero 必须保留标题、价值副标题和主 CTA 的连续阅读顺序，主图可以下移但不能挤压 CTA。若采用双按钮布局，小屏下可改成纵向堆叠。Hero 区如果没有真实产品截图，也可以使用抽象界面拼贴，但要避免廉价占位图破坏品牌感。
