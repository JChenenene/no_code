# 适用场景

该模式库适用于官网、发布页、案例页、定价页以及带多区块结构的单页网站，用来统一导航、页脚、区块顺序和卡片表单模块的组织方式。它适合解决“页面为什么看起来散、模块顺序为什么不顺”的问题。

# 页面目标

页面结构模式的目标是建立稳定节奏，让用户在浏览时始终知道当前位置、下一段内容和最终行动。导航负责降低跳转成本，页脚负责收束信息，分区顺序负责让信息从品牌到价值再到转化自然推进。

# 推荐信息架构

典型单页的推荐顺序是：导航、Hero、能力概览、流程或场景、案例或背书、FAQ 或补充说明、最终 CTA、页脚。导航项建议只保留 4 到 6 个高价值锚点，不要为了完整性塞太多入口。页脚内容则以品牌说明、常用链接、联系方式和版权信息为主，避免再次堆叠复杂营销内容。

# 必备模块

- 导航：品牌标识、核心锚点、主 CTA、移动端菜单
- 中段区块：能力卡片、流程说明、案例展示、FAQ
- 页脚：品牌简介、导航复用、联系入口、备案或版权信息
- 卡片模块：标题、摘要、状态、动作要清楚分层
- 表单模块：字段标签、辅助说明、按钮反馈统一

# 视觉与交互重点

导航应保持轻、稳、可读，不抢 Hero 视觉。滚动后可使用半透明吸顶或背景增强，但必须保证文字对比度。页脚不适合做过重视觉，重点在信息整洁和链接清晰。区块顺序要有轻重变化，例如用大标题区、小网格区、流程区、案例区形成节奏。卡片与表单模块需要统一圆角、边框、阴影和状态体系，避免同页出现多套样式语言。

# 文案语气与禁忌

导航命名应短且准确，例如“能力”“场景”“案例”“价格”“开始使用”。页脚文案以信息性表达为主，不要再次写大段宣传词。区块标题要说明作用，不建议使用太多抽象标题，例如“重新定义未来”这类无信息量措辞。

# 响应式与实现边界

移动端导航建议折叠为抽屉或全屏菜单，但主 CTA 应在展开前后都容易找到。页脚在小屏应转为简洁纵向结构，链接分组数量控制在用户可快速扫读的范围内。卡片与表单区块在窄屏下应优先单列展示，保持统一间距和清晰点击区域，不要勉强维持桌面端并排结构。
