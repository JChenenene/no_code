# 适用场景

该模式库适用于后台列表页、项目管理页、成员页、账单页、报表明细页等包含表格、筛选器和空状态的页面。它用来解决“列表信息很多，但不好找、不好筛、空了之后不知道怎么办”的问题，属于后台产品中复用频率非常高的一类结构规则。

# 页面目标

表格与筛选模式的目标是让用户能够快速定位目标数据，并在无结果、无权限、首次使用或筛选过严时获得明确下一步。高质量列表页不只是把数据排成行，而是让搜索、筛选、排序、分页和空状态都围绕同一个任务服务。表格若只强调展示而忽略操作路径，用户会在数据海中失去方向。

# 推荐信息架构

推荐顺序为：页头说明、搜索与筛选区、视图切换或批量操作区、数据表格主体、分页或加载更多、空状态反馈。筛选器要按用户决策顺序排列，最常用项放前面，次要高级筛选可折叠。空状态不应只有一句“暂无数据”，而应说明为什么为空，以及是否可以调整筛选、创建内容或联系管理员。

# 必备模块

- 搜索框：支持关键词搜索、回车提交、清空动作
- 快速筛选：状态、时间、标签、负责人、类型
- 表格主体：列标题、排序、状态、行级操作、批量选择
- 辅助反馈：加载中、空结果、错误重试、权限提示
- 空状态：首次创建引导、筛选无结果说明、行动按钮
- 分页与计数：当前结果数、总数、页码或滚动加载状态

# 视觉风格与交互重点

列表型页面要通过留白、列对齐和状态色建立秩序感。搜索与筛选区不能像一排没有优先级的控件集合，需有主次。交互重点在组合筛选、排序箭头、批量操作条、行 hover、状态提示和空状态 CTA。空状态尤其重要，不能每种场景都复用一张插图和一句通用文案，至少要区分“从未创建”“筛选无结果”“系统异常”三类。

# 文案语气与禁忌

表格页文案应清楚说明对象与动作，例如“筛选状态”“按更新时间排序”“暂无符合条件的项目”。空状态文案要给出下一步，如“调整筛选条件后重试”“创建第一个项目”“联系管理员开通权限”。避免使用模糊表达，如“这里什么都没有”，也不要在后台列表里加入与任务无关的情绪化文案。

# 响应式与实现边界

移动端表格通常需要降维为卡片列表、关键字段摘要或横向滚动表。筛选器在小屏更适合做成抽屉或底部面板，保证主内容区不被工具控件挤占。若没有真实数据源，演示态也应覆盖加载中、空结果、成功列表和错误重试几种典型状态，体现模式完整性。
