# 适用场景

该模板适用于业务分析中心、运营报表页、数据洞察页、项目指标看板和增长分析模块。它也可理解为“报表中心”“数据分析页”“指标洞察页”或“运营分析后台”。它比企业大屏更偏桌面端使用和深度阅读，强调筛选、对比、钻取和结论提炼，适合需要日常查看趋势和复盘数据的产品后台。

# 页面目标

报表中心页的目标是让用户快速完成“观察趋势、发现异常、下钻分析、导出结果”这条链路。页面既要提供概览，也要留出足够的细节空间支持进一步判断。与大屏不同，报表中心允许更高信息密度，但必须通过层级、筛选和模块组织，让用户始终知道当前正在看哪段时间、哪个维度和什么指标。

# 推荐信息架构

推荐结构为：顶部筛选栏、核心指标概览、趋势图与对比图、维度拆分区、明细表格、导出与订阅入口。常见筛选包括时间范围、业务线、项目、渠道、地区或成员。主图表应围绕最重要指标展开，下方再补充来源结构、转化漏斗、排行或异常明细。不要在一个页面塞太多同等重要的图表，否则用户会缺少分析主线。

# 必备模块

- 筛选栏：日期、维度、指标、对比周期、导出入口
- 概览卡：核心指标、环比同比、趋势方向、异常提醒
- 主图区：趋势折线、柱状图、面积图或漏斗图
- 拆分区：渠道、地区、产品线、成员等维度分析
- 明细表：排序、筛选、分页、导出、钻取入口
- 洞察补充：结论摘要、异常说明、推荐动作或订阅报表

# 视觉风格与交互重点

报表页需要比普通控制台更强调信息精度。图表颜色体系要统一，避免同屏出现太多饱和颜色和复杂图例。交互重点在筛选联动、时间范围切换、图表 hover、维度切换、表格排序和数据导出。用户必须能感知到每次筛选生效后的状态变化，如 loading、空数据、权限不足或延迟刷新提示。

# 文案语气与禁忌

报表页文案应以指标名称和分析动作命名，例如“最近 30 天活跃应用数”“按来源渠道查看转化率”“导出当前报表”。避免把运营分析界面写成宣传页面，也不要用过多抽象栏目名。若提供洞察摘要，应强调事实和建议，而不是情绪化评价。

# 响应式与实现边界

移动端报表中心应优先保留筛选器、核心指标和少量关键图表，复杂明细表可转为摘要卡片或横向滚动表。图表在小屏下需要简化图例和标签密度，确保仍可阅读。若没有真实数据接口，可使用可信模拟数据和静态图表占位，但筛选与状态切换逻辑应保持完整。
