# 适用场景

该模板适用于 AI 应用平台、低代码平台、内部工具平台和多项目协作系统中的应用列表页、项目管理页和工作空间首页。它也常被称为“项目列表页”“应用管理页”“工作空间项目页”或“我的应用页”。它用于承接“已经创建过内容的用户”，重点是让用户快速找到项目、理解状态并继续操作，而不是承担首次转化或营销展示任务。

# 页面目标

项目管理页的目标是帮助用户在最短时间内完成三件事：定位目标项目、判断当前状态、执行下一步操作。页面必须让“继续编辑、查看部署、筛选状态、创建新项目”这些高频动作足够突出。若信息过散，用户会在大量卡片或表格中失去方向；若功能过度收纳，常用操作又会被埋得过深。

# 推荐信息架构

推荐结构为：页头与主操作区、筛选与搜索区、项目列表主区域、状态与统计补充区、空状态或帮助区。项目列表可根据场景采用卡片视图、表格视图或二者切换。对于生成式产品，项目列表建议展示项目名称、类型、更新时间、生成状态、部署状态、最近操作人和快捷入口。若支持大量项目，筛选器应围绕状态、所有者、更新时间、标签或项目类型展开。

# 必备模块

- 页头区：标题、项目数量、主 CTA、最近视图切换
- 筛选区：搜索框、状态筛选、排序、标签筛选
- 列表区：项目名称、摘要、更新时间、状态、主要动作
- 批量操作：归档、删除、移动、导出或变更标签
- 空状态：首次创建引导、模板入口、帮助文档
- 侧边信息：统计概览、最近活动、协作提醒或资源配额

# 视觉风格与交互重点

项目页要在“信息密度”和“可读性”之间找到平衡。若使用表格，列宽、状态色和行级操作需保持规整；若使用卡片，卡片层级不能过重，避免整页像营销画册。交互重点在搜索联想、筛选联动、排序、批量选择、更多操作菜单和状态提示。项目状态必须一眼可辨，例如“草稿中”“生成完成”“部署中”“已归档”，不能只靠模糊颜色区分。

# 文案语气与禁忌

管理页文案要以对象和动作命名，像“新建应用”“继续编辑”“最近更新”“部署失败原因”这类表达比抽象标题更有效。不要在列表页使用过多情绪化欢迎语，也不要把每个空状态都写成大段教程。对危险操作要明确确认语义，对系统状态则要说明下一步，例如“部署失败，可查看日志后重新发布”。

# 响应式与实现边界

移动端项目管理页建议保留顶部主操作、搜索和项目卡片列表，复杂表格字段可折叠为次级信息。批量操作在小屏应转为长按或多选模式，不建议强行保留桌面端多列布局。若演示态没有真实项目数据，应提供多种可信状态的模拟记录，让用户能判断页面结构和交互逻辑是否完整。
