# 适用场景

该模板适合生成产品工作台首页、管理控制台首页、团队协作入口页和业务后台总览页。它更强调信息结构、操作效率和状态感知，不适用于纯展示型营销页面。

# 页面目标

控制台首页的核心是让用户一进入就知道“当前状态如何、最重要的待办是什么、接下来应该点哪里”。页面需要清楚区分数据总览、快捷操作、任务状态、项目列表和系统通知，不要把所有内容堆成一页长卡片流。

# 推荐信息架构

推荐结构为：顶部工具栏、概览指标、核心任务区、项目或应用列表、最近活动、通知或帮助区。对于 AI 生成平台，概览指标可以围绕应用数量、最近生成记录、部署状态、协作成员、剩余额度等展开。工作台首页需要给出至少一个可执行的首要动作，例如“新建应用”“继续编辑”“查看部署”。

# 必备模块

- 顶部工具栏：搜索、通知、个人菜单、主操作按钮
- 概览指标：3 到 5 个关键指标，数字层级清晰
- 快捷操作：新建、导入、继续最近项目、查看日志
- 项目列表：名称、状态、更新时间、主要动作
- 最近活动：生成、修改、部署、失败记录
- 辅助区：通知、使用建议、帮助入口或引导卡

# 视觉与交互重点

后台首页视觉应克制、有秩序，优先使用明确网格和稳定留白。指标卡、列表、状态标签和快捷按钮需要建立统一层级，不要让所有模块都抢视觉焦点。交互重点在筛选、搜索、表格行操作、状态标签和 hover 提示。若有图表，应以快速读数为目标，不要过度装饰。

# 文案语气与禁忌

后台文案要短、可操作、系统化，优先使用“新建应用”“继续编辑”“查看失败原因”“最近 7 天生成量”等表达。避免首页出现营销横幅式大段话，也不要在关键操作区堆叠解释性说明。状态文案必须准确，例如“部署中”“已发布”“生成失败”“等待输入”。

# 响应式与实现边界

移动端后台首页应优先保留概览指标、核心待办和项目列表，复杂表格与多列图表要折叠或转为卡片。侧边导航在小屏可改为抽屉式，但主操作按钮必须可快速触达。如果页面没有真实数据源，演示态应使用可信的模拟数据，不要出现空白控制台。
