# 适用场景

该模板适用于预约演示页、需求收集页、项目咨询页、线索留资页等高转化表单页面。它通常用于官网 CTA 的后续承接页，重点是把用户从“有兴趣”顺利带到“愿意提交信息”。

# 页面目标

表单页的目标是用尽量少的阻力收集足够信息，并让用户理解提交后的下一步。页面需要回答三个问题：为什么现在填写、需要填写什么、提交后会发生什么。不要让表单页变成另一个内容很重的营销页，也不要让表单本身长到让人失去耐心。

# 推荐信息架构

推荐采用双栏或单聚焦布局：一侧说明预约价值、服务流程和可信背书，另一侧放表单。若表单字段较少，可采用单列居中布局并强化标题与提交后预期。页面顺序通常为：标题说明、表单主体、服务承诺、FAQ 或隐私说明。

# 必备模块

- 标题区：预约价值、适用对象、预计沟通内容
- 表单字段：姓名、公司、联系方式、需求类型、项目描述
- 提交说明：预计响应时间、沟通方式、是否可试用
- 信任补充：客户类型、交付经验、隐私承诺
- FAQ：字段用途、是否收费、提交后流程

# 视觉与交互重点

表单页要把输入体验放在第一位。输入框、下拉框、单选项、文本域、提交按钮的层级必须清晰，错误状态和成功状态要统一。可使用轻量品牌视觉，如背景渐变、品牌纹理、简洁插图，但不能让视觉元素干扰表单填写。提交按钮应有加载、禁用和成功提示，不可点击后无反馈。

# 文案语气与禁忌

文案应建立信任而不是施压，例如“留下需求，我们将在 1 个工作日内联系你”“可先描述当前项目阶段与目标”。避免使用强迫型表述或夸张承诺，如“马上抢占红利”“不填就错过机会”。字段标签要清楚，辅助说明尽量写在字段下方而不是藏在很远的位置。

# 响应式与实现边界

移动端表单需使用单列布局，确保输入框高度、按钮宽度和键盘弹起后的可操作性。长文本域、协议勾选、提交反馈必须在小屏下完整可见。若没有真实提交接口，前端演示态也应提供校验、成功提示和重置逻辑，让页面在预览中成立。
