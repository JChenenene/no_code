# 适用场景

企业数据大屏适用于经营总览、运营监控、项目进度驾驶舱、销售战情室、产线监测等需要“快速扫视 + 强状态感”的场景。它不同于普通后台首页，重点是大视距浏览和指标关联，而不是逐条操作。

# 页面目标

大屏的目标是帮助用户在远距离或短时间内判断系统整体状态、关键指标波动和异常位置。页面首先要建立清晰的主指标，再安排趋势、结构、地域或进度类信息。大屏必须能在静态截图状态下也读懂，不能依赖交互才能理解内容。

# 推荐信息架构

推荐采用三段式布局：顶部总标题与时间状态，中部核心指标与主趋势，下部结构分布与明细补充。常见方式是左右辅助、中间聚焦，或者上总览下细分。每个区域最好只承担一种信息职责，例如中间放总量和趋势，左侧放来源结构，右侧放进度或告警。

# 必备模块

- 顶部信息：标题、当前时间、数据更新时间、全局状态
- 核心指标：总量、同比、环比、完成率、告警数
- 主趋势图：按日、周、月变化
- 结构图：行业、区域、产品线、渠道分布
- 进度或排行：Top 列表、项目完成度、站点排名
- 异常提示：告警数量、异常点位、风险摘要

# 视觉与交互重点

大屏适合深色底、高对比荧光色、高亮边框和统一图表风格，但要克制使用发光特效。信息优先级要靠字号、色彩和区域大小体现，不能只靠颜色区分。图表应采用统一的坐标、图例和标注规则，数字必须大且清晰。若提供切换交互，应是时间粒度或指标视角切换，避免复杂表单交互。

# 文案语气与禁忌

标题和指标名称必须标准、准确、短，适合远距离阅读，例如“本月签约额”“在线项目数”“异常工单数”。不要使用营销型标题，也不要在图表旁堆很多说明文字。大屏更需要数据标签和状态说明，而不是长段描述。

# 响应式与实现边界

如果同一页面需要兼顾桌面和大屏预览，应设计为桌面端正常浏览、大屏端按比例放大的双适配方案。小屏设备不能照搬整套大屏布局，应改成纵向信息摘要页。若没有真实图表库或后端数据，需使用可信模拟数据和静态图表容器，保证结构完整可演示。
