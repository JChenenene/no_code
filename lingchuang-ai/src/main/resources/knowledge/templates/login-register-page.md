# 适用场景

该模板适用于 SaaS 产品登录页、注册页、统一认证入口页，以及需要兼顾品牌感和表单转化的账户系统页面。它不只是一个简单表单壳子，而是“品牌信任 + 输入效率 + 操作反馈”的组合界面。

# 页面目标

登录注册页的主要目标是降低进入门槛，让用户快速完成认证，同时保持产品品牌的一致性。页面需要明确区分登录与注册、主流程与次流程、必填与可选项，并通过简洁反馈减少错误输入和焦虑感。

# 推荐信息架构

推荐布局为左右或上下双区：一侧用于品牌、价值或辅助说明，一侧用于表单。登录页优先展示邮箱或账号、密码、忘记密码、主按钮和第三方登录入口；注册页则增加名称、密码规则、协议确认、试用说明。不要在认证页塞入过多营销模块，以免打断操作。

# 必备模块

- 品牌区：Logo、简短定位语、受益说明或可信背书
- 表单区：输入框、按钮、错误提示、辅助链接
- 模式切换：登录与注册入口切换明确
- 次流程：忘记密码、第三方登录、验证码登录按优先级排列
- 协议与隐私：注册态必须清晰可见
- 成功或失败反馈：按钮状态、校验提示、提交中状态

# 视觉与交互重点

视觉上要保证输入区域清晰、对比度足够、焦点态明确。输入框、按钮、错误提示、成功提示要有统一状态体系。深色科技风可以保留，但表单区域必须比背景更稳定、更容易阅读。交互重点在输入聚焦、错误提示、提交 loading、密码可见切换和模式切换，不做无意义动画。

# 文案语气与禁忌

认证页文案应简短清楚，按钮文本直接用“登录”“创建账号”“继续使用”，错误信息要告诉用户该怎么改，而不是只说失败。避免在认证页出现过长宣传文案、课程型提示、过度夸张的欢迎语，也不要让安全提示写得像技术白皮书。

# 响应式与实现边界

移动端优先单列布局，品牌区简化成 Logo 与一句定位语即可。输入框、按钮和链接之间要留足触控空间，避免误触。若后端接口未接入，演示态也要提供前端校验和提交反馈，不可直接静默无响应。验证码、倒计时、协议勾选等元素需要在小屏下保持完整可见。
