# 适用场景

该模板适用于 SaaS 产品首次使用流程、应用创建向导、组织初始化、数据接入配置和部署前准备流程。它也常被直接搜索为“新手引导页”“创建向导页”“步骤流程页”或“首次配置页”。它特别适合“用户第一次进入系统，需要在几个明确步骤内完成关键设置”的场景，也适用于复杂功能的分步创建界面，而不是简单单表单提交页面。

# 页面目标

创建向导页的目标是降低第一次使用的心理负担，让用户明确知道当前在第几步、还剩几步、每一步的作用是什么，以及完成后会得到什么结果。页面不应让用户在一个屏幕里面对过多字段，而应通过分段决策和持续反馈，帮助用户稳定推进。理想体验是“每一步都清楚、可回退、可继续、不会迷路”。

# 推荐信息架构

推荐结构为：顶部标题与步骤说明、步骤导航或进度条、当前步骤主内容区、侧边摘要区或帮助区、底部操作栏。若流程较短，可使用水平步骤条；若步骤较多，可使用左侧纵向步骤导航。每一步只承担一个主要任务，例如填写基本信息、选择模板、连接数据源、设置风格、确认发布。不要把总结页提前塞进中间步骤，也不要让用户在前几步就填写后续才用到的大量高级配置。

# 必备模块

- 步骤导航：显示当前步骤、已完成步骤和剩余步骤
- 主表单区：当前步骤所需输入、选择器、上传区或开关项
- 侧边摘要：当前配置概览、预计效果、示意图或帮助说明
- 操作栏：上一步、下一步、保存草稿、取消、完成
- 校验反馈：字段错误、缺失项提醒、步骤级阻塞提示
- 完成页：创建结果、下一步建议、直接进入控制台或继续配置

# 视觉风格与交互重点

创建向导需要比普通后台页更强调引导感，但不能做成过于娱乐化的流程动画。视觉上建议用清晰的分区、稳定的步骤条、适度高亮的当前步骤，以及明显的底部操作栏。交互重点在步骤切换、字段即时校验、返回上一步后的状态保留、保存草稿和完成反馈。若存在上传、连接测试或环境校验，必须提供处理中状态和结果说明，不能让用户猜测系统是否在工作。

# 文案语气与禁忌

向导页文案应使用指令式但友好的表达，例如“选择一个基础模板开始”“填写团队名称后可稍后再修改”“连接成功后即可进入下一步”。不要把步骤标题写成抽象概念，更不要在字段下方堆积过长教程说明。对必须操作的内容要明确原因，对可选项则说明何时再配置即可，避免制造不必要的决策压力。

# 响应式与实现边界

移动端向导页应优先保留当前步骤标题、核心输入区和底部操作按钮，复杂摘要区可以折叠为抽屉或概览卡。步骤导航在小屏可简化成“第 2 步 / 共 5 步”的形式，但当前状态必须始终可见。若没有真实后端流程，演示态也要保留步骤流转、表单校验、草稿保留和完成页反馈，确保流程体验成立。
