# 适用场景

定价页适用于 SaaS 产品套餐说明、版本权益对比、团队版与企业版转化页等场景。它的重点不是把所有细节列满，而是帮助用户快速判断“我适合哪个版本、差异在哪里、如何继续行动”。

# 页面目标

定价页要降低决策摩擦，让用户在短时间内完成套餐理解、权益比较和下一步行动。页面要清楚呈现版本层级、核心限制、升级路径和常见问题，避免用户为了看懂价格结构还要反复跳转其他页面。

# 推荐信息架构

推荐顺序为：价格总览、套餐卡片、权益对比表、适用对象说明、FAQ、企业咨询 CTA。价格总览区最好先说明计费周期、是否支持试用、是否含部署或团队协作。权益对比要聚焦对选择真正有影响的项，而不是把不重要字段堆成超长表。

# 必备模块

- 计费切换：月付、年付、联系销售等
- 套餐卡片：基础版、专业版、团队版、企业版
- 重点差异：项目数量、生成额度、协作人数、部署方式、支持级别
- 推荐标识：标出最适合大多数用户的版本，但不要强行诱导
- 对比表：只保留关键权益，信息密度要可控
- FAQ：价格、退款、试用、升级、发票、私有部署
- 企业 CTA：联系销售、预约方案咨询、获取报价

# 视觉与交互重点

定价页需要高可读性和高对比度，不要为了追求炫技牺牲信息清晰度。套餐卡片可以有主推荐态、描边高亮和轻微放大，但不能让所有卡片视觉权重一样。计费切换、套餐选择、FAQ 折叠是主要交互点；对比表应有固定列标题或移动端折叠策略，避免难以阅读。

# 文案语气与禁忌

价格描述要透明、简洁、直接，优先说明“适合谁”和“买到什么”。不要使用带压迫感的营销表达，如“错过就没有”“限时最后机会”之类措辞。推荐语应强调场景适配，例如“适合需要稳定交付官网和运营页的小团队”，而不是纯销售导向的空话。

# 响应式与实现边界

移动端定价卡片建议纵向堆叠，并把推荐版本置顶或高亮。对比表在小屏应转为“按权益分组 + 套餐逐项对比”的结构，不要保留桌面端大表格原样缩放。如果没有真实支付能力，按钮文案应指向试用、预约演示或咨询，而不是伪造结算流程。
