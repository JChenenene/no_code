# 适用场景

该模板适用于 SaaS 产品的单页落地页、活动承接页、流量投放页和以转化为导向的专题页。它比官网首页更聚焦，比产品详情页更强调行动路径，适合“一个核心卖点 + 一个主要转化目标”的页面任务，例如获取试用、引导预约、承接广告或集中讲清某个功能方案。

# 页面目标

落地页的目标不是面面俱到，而是把用户从“看到信息”迅速带到“愿意采取行动”。页面应在较短路径内完成价值表达、信任建立和 CTA 收束，让用户在极少的滚动内就知道产品适合谁、能解决什么问题、现在应该点击什么按钮。若一个页面同时承担太多产品线或太多业务线说明，转化效率通常会明显下降。

# 推荐信息架构

推荐顺序为：导航弱化版、Hero 首屏、核心卖点区、结果或收益区、工作流程或使用步骤、社会证明、FAQ、最终 CTA。与首页不同，落地页的导航可以更轻，甚至只保留品牌标识和一个返回主站入口。首屏之后应直接进入卖点和收益，不建议把品牌故事、团队介绍、公司历程之类与转化关系较弱的内容排在前面。

# 必备模块

- 首屏：清晰标题、结果导向副标题、主 CTA、辅助说明
- 卖点区：3 到 4 个高价值能力点，强调具体收益
- 收益区：速度、成本、效果、协作效率等可感知结果
- 流程区：从开始使用到看到结果的步骤说明
- 证明区：客户标识、关键数字、案例摘要、评价引用
- FAQ：回答常见顾虑，例如是否需要部署、是否支持定制、多久可上线
- 结尾 CTA：再次收束动作，不再增加新信息负担

# 视觉风格与交互重点

落地页应比控制台页面更有情绪，但又不能像纯视觉海报那样缺乏信息承接。首屏可使用更强的主视觉、光效、渐变和场景化界面拼贴，但中段必须回到清晰的信息块结构。CTA 按钮应有稳定主次关系，主按钮使用实体高对比色，次按钮可以用描边或文字链接。交互重点在滚动节奏、CTA hover、数字或卡片的轻量强化，不要把每个区块都做成强动画舞台。

# 文案语气与禁忌

文案应直接说明对象、问题和结果，例如“为 AI 产品团队快速生成品牌级站点”“把需求收集到上线交付压缩到更短周期”。落地页尤其忌讳空话堆砌和形容词过载，像“重新定义未来”“革命性体验”这类措辞会削弱可信度。按钮用词要有动作感，如“立即体验”“预约演示”“获取方案”，避免泛化成“查看更多”。

# 响应式与实现边界

移动端落地页必须优先保证首屏标题、副标题和主 CTA 的完整阅读链路，不能让大图或装饰图挤压可点击区域。信息区块在小屏下建议转为单列，保留清晰段落与足够留白。若页面用于前端演示，CTA 和表单需要提供可理解的假交互反馈，不能出现点击后完全无响应的空状态。
