# 适用场景

该模板适用于模板中心、场景库、组件案例库、作品画廊和“从模板开始创建”的浏览页。它也经常被叫作“模板库”“案例画廊页”“模板市场首页”或“模板浏览页”。它常见于 AI 生成平台、低代码平台、设计系统站点和模板市场，承担“让用户先看见可能性，再开始使用”的任务。

# 页面目标

模板中心页的目标是帮助用户快速理解平台能生成什么、不同模板适合什么场景，以及如何从浏览自然转入创建。它不是简单的卡片堆砌页，而是一个兼顾探索和转化的内容入口。页面需要让用户能按行业、页面类型、风格或目标快速筛选，并在卡片层级上直接看到模板核心特征。

# 推荐信息架构

推荐结构为：首屏说明与搜索、分类与筛选区、模板卡片网格、精选模板或专题区、使用流程说明、结尾 CTA。若模板数量很多，可加入排序、收藏、最近使用或推荐标签；若数量有限，则更适合强调专题策展和精选推荐。卡片区不应只是缩略图拼盘，至少要展示模板名称、适用场景、风格标签和主要动作。

# 必备模块

- 首屏区：页面目的说明、搜索框、主 CTA
- 筛选区：行业、页面类型、风格、技术栈、是否适合新手
- 模板卡片：缩略图、名称、摘要、标签、操作按钮
- 专题区：热门模板、编辑精选、最近上新、适用场景合集
- 说明区：如何使用模板、如何二次编辑、是否支持导出部署
- 结尾 CTA：开始创建、查看全部案例、提交定制需求

# 视觉风格与交互重点

画廊页可以比普通后台更强视觉一些，但必须控制信息噪音。模板缩略图要建立统一裁切和边框规则，卡片 hover 可增强预览感，但不要导致整页闪动。交互重点在筛选、搜索、卡片快速预览、收藏、分页或无限滚动，以及模板详情入口。精选模板应在布局上明显区别于普通卡片，避免所有内容一个权重。

# 文案语气与禁忌

模板名称和摘要应帮助用户快速判断适用性，例如“高端科技感 AI 官网首页”“企业数据大屏总览模板”“预约演示表单页”。避免使用过度抽象的命名，也不要让卡片摘要只剩空泛风格词。CTA 文案可以直接使用“立即套用”“查看详情”“从此模板开始”，不要把关键动作写得含糊不清。

# 响应式与实现边界

移动端模板中心应优先保留搜索、筛选入口和卡片列表，复杂多列网格改为双列或单列布局。筛选器适合收纳为底部抽屉或顶部弹层，不建议在小屏强行保留侧栏。若没有真实模板数据，可使用静态卡片和模拟标签，但缩略图、状态与操作反馈仍要像真实产品。
