# design-ui

## when_to_use

- 用户要求页面、美化、视觉风格、响应式布局或交互体验。
- 生成结果需要面向真实用户展示，而不是仅验证功能。

## instructions

- 优先保证首屏信息清晰、视觉层次稳定、移动端不溢出。
- 选择克制的配色和合理留白，不使用无意义装饰。
- 按页面目的组织内容，避免营销文案堆砌。
- 所有按钮、链接、输入状态都要有清晰反馈。

## examples

- 个人介绍页需要展示姓名、身份、简介和联系方式。
- 管理后台需要优先支持快速扫描和重复操作。
